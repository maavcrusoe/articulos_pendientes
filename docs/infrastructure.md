# Infrastructure Changes

This document records infrastructure, deployment, and operations changes related to the project.

## Purpose
- Provide a concise, discoverable record of any new infrastructure components, deployment changes, or operational configuration added to the project.
- Link code/config changes to runbooks and environment variables.

## When to update
Update this file (or add a new dated entry) when you:
- Add or modify containers, Dockerfiles, docker-compose services, or Kubernetes manifests.
- Add cloud resources (e.g., load balancers, storage buckets, managed DBs) or IaC templates.
- Change CI/CD pipelines, deployment scripts, or add new runner agents.
- Introduce a new external dependency that requires infra changes (e.g., Redis, RabbitMQ).

## Required information for each change
- **Title:** short descriptive title
- **Date:** YYYY-MM-DD HH:MM (UTC recommended)
- **Author:** GitHub username or name
- **What changed:** concise summary of files and services modified/added
- **Affected files / paths:** list of repo paths (Dockerfile, terraform/, k8s/, docker-compose.yml)
- **New environment variables:** names and short purpose (do NOT include secrets)
- **Deployment steps / roll-back:** commands or links to runbook
- **Security considerations:** firewall rules, public exposure, secret storage
- **Notes / links:** links to external consoles, tickets, PRs
